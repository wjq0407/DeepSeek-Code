# DeepSeek Code Agent · 分阶段学习路线

> 目标：让你**真正理解**自己做的这个项目，而不是背结论。
> 用法：每个阶段先读「核心概念」+「关键文件」，再用「自检问题」检验自己，最后做「小练习」。
> 风格：费曼法（一句话本质 + 生活类比）+ 苏格拉底（自检提问逼你推导）。

---

## 一句话本质（费曼开场）

**你的项目 = 一个用 DeepSeek 模型当"大脑"、用 TypeScript 写的"循环外壳（harness）"，让模型能反复"思考→调工具→看结果"直到把编程任务做完的命令行智能体。**

类比：模型是司机（会开车、认路），harness 是车和路（油门、刹车、仪表盘、护栏）。你写的是车，不是司机。

## 阶段 ① 心智模型：Agent 是什么

**核心概念**
- **Agent Loop 不变式**：`while (模型还要用工具) { 模型思考 → 调工具 → 把结果喂回去 }`，循环永不变，变的是循环外的机制（权限、压缩、记忆）。
- **Model + Harness 二分**：Agency（主动性）来自模型训练；你能改的是 harness（外壳）。
- **和传统程序的区别**：传统程序是你写死 if/else 逻辑；Agent 是让模型在循环里**自己决定下一步做什么**。

**关键文件**：`README.md`（看定位）、`src/agent/loop.ts`（看那个 while 循环长啥样）。

**自检问题**
1. 为什么叫"循环"而不是"一次函数调用"？
2. 如果模型一直说"我还要调工具"但什么都不做，会发生什么？（提示：early-exit）
3. 你写的 harness 和模型本身，职责边界在哪？

**小练习**：用一句话向"完全不懂编程的家人"解释你的项目。

## 阶段 ② 主循环：推理流程

**核心概念**
- 一次交互 = 系统提示词 + 历史 + 用户消息 → 模型返回 → 若有 `tool_use` 则执行工具 → 把结果追加回历史 → 再问模型 → 直到 `stop_reason` 不再是工具调用。
- 工具调用机制：模型输出结构化 tool_use（名字+参数）→ harness 查注册表 → 执行 → 返回 `tool_result`。
- `temperature` 压低（0.1）保工具选择确定性。

**关键文件**：`src/agent/loop.ts`、`src/tools/index.ts`。

**自检问题**
1. 模型怎么"知道"有哪些工具可用？（系统提示词里的清单）
2. 工具结果回灌前为什么要截断？（上下文预算）
3. 思考内容（reasoning）和工具调用，在协议里怎么区分？

**小练习**：在纸上画出"用户提问 → 模型 → 工具 → 回灌"的时序箭头。

## 阶段 ③ 上下文与记忆：长任务的命门

**核心概念**
- 上下文窗口有限，长任务会"爆窗"。
- **两级压缩**：`snipText`（head+tail 确定性截断）+ `compact`（每轮末自动压缩）+ 仍超预算才硬丢旧条。
- **双层记忆 RAG-lite**：用户级 `~/.dsa/memory` + 项目级 `<cwd>/.dsa/memory`；DeepSeek 无 embedding 端点 → 本地 BGE 模型，无模型降级关键词。记忆只服务"非代码语义"，不进 grep 工具链。

**关键文件**：`src/context/history.ts`、`src/memory/manager.ts`。

**自检问题**
1. 为什么不直接用向量数据库做记忆？（提示：无 embedding 端点、离线、轻量）
2. snip 和 compact 的区别？（确定性裁剪 vs LLM 摘要）
3. 为什么记忆要分"用户级/项目级"两层？

**小练习**：设想一个跑 50 轮的任务，画上下文窗口随时间的变化曲线。

## 阶段 ④ 工具与技能：你的差异化能力

**核心概念**
- **复合工具二次推理**：`review_code`/`audit_dependencies`/`git_commit_msg` 内部再调一次 `complete()`（不带 tools，防循环），用真实上下文做专精判断。
- **6 个差异化工具**：review_code / audit_dependencies / git_commit_msg / terminology / project_discover / delegate（前 3 个是复合工具）。
- **技能双作用域 + 白名单**：项目级 `.workbuddy/skills/` 覆盖全局 `~/.workbuddy/skills/`；全局白名单 `skills.allow.json` 防污染。

**关键文件**：`src/tools/review.ts`、`src/skills/loader.ts`、`src/tools/use_skill.ts`。

**自检问题**
1. 复合工具为什么"内部再调一次模型"而不是让主模型直接做？
2. 子提示为什么必须含真实上下文？（否则幻觉）
3. 全局技能默认全开会有什么问题？

**小练习**：挑一个差异化工具，写一段"它比让主模型直接做好在哪"的口播。

## 阶段 ⑤ 工程化质量：区分玩具与产品

**核心概念**
- **双模型 V4 路由**：`deepseek-v4-flash`（主循环，关思考保速度）+ `deepseek-v4-pro`（复合工具，开思考保质量）+ 成本估算。
- **测试**：TS strict 零 any、零依赖 golden cases（23 个）、先验证评测 harness 可信再信结果。
- **权限三模式** + **写前 diff 审批**（create/edit/delete 的 preview 不落盘确认）。
- **四路 early-exit**：全失败/重复各 3 轮即停，防死循环空转。

**关键文件**：`src/llm/deepseek.ts`、`test/`、`src/permissions.ts`、`src/agent/loop.ts`（early-exit 段）。

**自检问题**
1. 为什么主循环用 flash、复合工具用 pro？（速度 vs 质量权衡）
2. 写前 diff 审批解决了什么风险？
3. 没有 early-exit 会有什么事故？

**小练习**：给"双模型成本"算一笔账（假设 1 任务 = 10 轮主循环 + 3 次复合）。
